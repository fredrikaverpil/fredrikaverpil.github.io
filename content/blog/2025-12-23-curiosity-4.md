## LLMs

[Gemini 3 Pro Preview](https://deepmind.google/models/gemini/pro/) was released,
but I prefer [Claude Opus 4.5](https://www.anthropic.com/news/claude-opus-4-5)
in [Claude Code](https://claude.com/product/claude-code) over Gemini for
anything coding related. I've also started to rely more and more on skills,
which I heard
[OpenAI is adopting too](https://simonwillison.net/2025/Dec/12/openai-skills/).
But for everything else, it's Gemini 3 all the way. I wonder if
[Gemini 3 Flash](https://deepmind.google/models/gemini/flash/) will beat Claude
in coding tasks (it kind of
[sounds like some people are very impressed](https://news.ycombinator.com/item?id=46301851))?

I also helped set up LLM access via
[Vertex AI in Google Cloud](https://cloud.google.com/vertex-ai). This means
using Claude and Gemini models, professionally! ðŸ¤  I even hacked Claude into

Speaking of juggling all these CLIs; Claude Code, Gemini CLI, Copilot CLI (and
OpenCode, which I don't really use much anymore)... The
[sidekick.nvim](https://github.com/folke/sidekick.nvim) is my go-to way to
interact and switch between them. I can highly recommend it before any
[ACP-integrated](https://agentclientprotocol.com/overview/introduction) Neovim
plugin.

A big one for me personally this autumn was to finally get streaming results
implemented into
[neotest-golang](https://github.com/fredrikaverpil/neotest-golang) and get
v2.0.0 out the door.

[nvim-treesitter](https://github.com/nvim-treesitter/nvim-treesitter) finally
switched officially over onto its `main` branch, which puts the debate to rest
whether to depend on the legacy `master` branch or the new one.

For [godoc.nvim](https://github.com/fredrikaverpil/godoc.nvim) I implemented a
tree-sitter parser which identifies `go doc` parts that I want
syntax-highlighted as Go code. Check it out, it's pretty neat!

From my personal stash of UX improvement scripts, I bring you Go symbols
searching across all files in the current package, with
[this custom snacks.nvim picker](https://github.com/fredrikaverpil/dotfiles/commit/407542a502433959c85fea61d80d74b945b08104)
(check out the main branch for any touchups).

Then I noticed Anton Zhiyanov has a new
[Accepted! blog series](https://antonz.org/tags/accepted/), which talks in depth
around accepted Go proposals which have yet to hit the language. Super nice with
all these code examples. If you rather want to track this yourself, subscribe to
[this never-ending GitHub issue](https://github.com/golang/go/issues/33502).
